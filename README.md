# 项目概述
vue2-esayGO 是基于 Vue 2 构建的电商类移动端项目，聚焦于电商核心业务场景的前端实现，涵盖商品展示、购物车、订单管理等基础电商模块（可根据实际业务补充）。项目采用 Vue 生态主流技术栈，遵循前端工程化规范，具备可维护性、可扩展性的特点，适用于电商业务前端开发场景的技术验证与落地。

## 技术栈

### 核心框架
Vue 2：项目核心前端框架，用于构建用户界面，利用vant-ui组件库、响应式特性提升开发效率与页面交互体验。

### 工程化工具
1.Vue CLI：项目脚手架工具，提供项目初始化、开发调试、打包构建、代码检测等一站式工程化能力。

2.npm：包管理工具，负责项目依赖的安装、管理与版本控制。

3.Babel：JavaScript 编译器，实现 ES6+ 语法向低版本浏览器兼容的转换。

4.PostCSS：CSS 处理工具，实现 CSS 语法增强、自动前缀补全等能力。

## 生态配套
1.Vue Router：前端路由管理，实现页面间的路由跳转、路由守卫、路由懒加载等功能。

2.Vuex：状态管理库，统一管理项目中全局共享状态（如用户信息、购物车数据、商品列表状态等）。

3.Mixins：Vue 混入机制，抽离通用逻辑，提升代码复用性。

4.utils封装axios，统一管理拦截器，减少维护成本，优化代码结构

## 项目亮点
1.组件化开发：将页面拆分为通用组件（如商品卡片、分页、弹窗）和页面级组件，降低耦合度，提升复用性。

2.状态管理：基于 Vuex 实现全局状态（如购物车数据、用户信息）的统一管理，保证状态流转清晰可追溯。

3.接口封装：api 目录统一封装接口请求，结合拦截器实现请求 / 响应的统一处理（如请求头添加 token、响应错误统一提示）。

4.工具函数抽离：utils 目录封装通用工具（如时间格式化、数据校验、本地存储），提升代码复用率。

5.路由优化：采用路由懒加载（import() 方式），减少首屏加载体积，提升页面加载速度。

6.样式规范化：通过全局样式 + 组件样式隔离（scoped），避免样式冲突，提升样式维护性。
